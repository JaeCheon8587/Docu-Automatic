# Docusaurus 문서 자동화 파이프라인

팀 내 제품 레포의 git push 시 AI가 기술 문서를 자동 생성하여 Docusaurus 사이트로 빌드/배포하는 파이프라인. 대상 언어: C++, C#, JavaScript, Python.

## 전체 흐름

```
git push (제품 레포) → CI → AI .md 생성 (테마 순회) → docs-auto 브랜치 → 중앙 배치 빌드 → Docusaurus 배포
```

## 아키텍처 요약

> 상세: [아키텍처](02.아키텍처.md) · 설계 근거: [설계 결정사항](01.설계-결정사항.md)

**1단계 오케스트레이션** — Main CLI가 테마 루프를 직접 관리하고, 각 단계를 Level 1 Agent에 위임한다.

```
Main CLI (Level 0)
  ├─ scout    : 탐색 + 판단 + 요구사항서 작성
  ├─ docu-writer : md 초안 작성
  └─ critic   : 독립 검증
```

**테마별 4단계 사이클**:

```
scout 판단 → docu-writer 작성 → critic 검증 → Main 저장
         ↑                          │ FAIL
         └──── docu-writer 재시도 ←──┘ (최대 2회)
```

| 정책 | 내용 |
|------|------|
| Full Reset | 매 테마마다 scout/docu-writer/critic 신규 생성 (이전 에이전트 resume 금지) |
| 재시도 | 검증 실패 시 docu-writer 재호출 최대 2회, 초과 시 `auto_generated_warning` 태그 |
| 중단 조건 | 3개 테마 연속 FAIL 시 파이프라인 중단 |
